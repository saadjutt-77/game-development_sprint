# game-development_sprint
Modeling  sprint 1

In this phase, I designed the structure and functionality of the mini multiplayer game. I created the game flow, user interface layouts, gameplay mechanics, player interactions, and database/network architecture. The design helped define how players would connect, play, and communicate within the game.

Implementation Phase   sprint 2

In this phase, I developed the actual game by converting the design into code. I implemented the game screens, multiplayer networking features, player movement, game logic, user authentication, and other core functionalities using the selected game development framework and programming language.

Testing Phase sprint 3

In this phase, I tested the game to identify and fix bugs, errors, and performance issues. Multiplayer connectivity, gameplay mechanics, user interface, and overall functionality were tested to ensure a smooth gaming experience.

Deployment Phase Sprint 4

In this phase, I prepared and released the game for users. The application was built, configured, and deployed to the target platform so that players could access and play the game.

Maintenance Phase  Sprint 5

In this phase, I monitor the game after deployment, fix any issues reported by users, improve performance, add new features, and provide updates to enhance the overall gaming experience.
